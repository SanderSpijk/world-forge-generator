VERSION 0.9.1 SUMMARY
================================================================================
This update captures the latest improvements to the WorldForge module:

1. Buildings.json Data Consolidation
   - Fixed incomplete buildings.json (dak, hoogte, drukte, detail_inn,
     detail_shop, sfeer, bouwmateriaal arrays were empty)
   - Filled with generic fallback items (6 roof types, 5 heights, 6 activity
     levels, 7 inn details, 6 shop details, 5 materials)
   - Shop/Inn/Tavern descriptions now render correctly

2. Section Header Translations
   - Added 30+ missing translation keys to nl.json and en.json
   - Inn/Tavern: Staff, Guests, Gossip, room types (Single, Double, Group,
     Dormitory, PerNight, PerBed)
   - Criminal Organization: Operations, Quirk, Services, Officers, all
     Readout keys
   - Faction: All Readout keys (IsA, OperatesFrom, Entrance, Resource, Goal,
     Territory, OpStyle, Rival, Ally)

3. House/Family Generator Enhancements
   - Family members now render with wfNpcItem() UI component
   - Integrated action buttons: Create Actor (🎭) and Campaign Codex (📖)
   - Family composition moved to header as badge (alongside LARGE/WEALTHY)
   - Backstory now uses wfReadout() layout (blockquote with gold border)
   - Removed ComfyUI debug prompt display

4. District House Button System
   - Implemented houseType attribute in cities.json (matches shop/POI pattern)
   - Buttons appear in residential districts:
     • Noble District: villa, coach_house
     • Middle Class: house, large_house
     • Lower Class: small_house
     • Slums: shack, tent
   - Event handler in worldforge-app.js for house generation to chat

5. Complete i18n Coverage
   - All section headers, readouts, and labels now have translation keys
   - No more untranslated "WF.*" keys visible to users
   - Consistent formatting across all generators
